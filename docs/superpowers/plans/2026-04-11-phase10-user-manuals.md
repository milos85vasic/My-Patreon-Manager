# Phase 10 — User Manuals Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce extended, step-by-step user manuals covering every user surface: end-to-end walkthrough (zero to first published Patreon post), per-CLI-subcommand manual, admin manual, developer manual, deployment manual for every supported target.

**Architecture:** Manuals live under `docs/manuals/`. Each manual is a self-contained document with prerequisites, numbered steps, expected output per step, screenshots generated by a headless browser script, and a troubleshooting section at the end. Examples inline-tested via Phase 9 docs-example harness.

**Tech Stack:** Markdown, chromedp (for screenshot generation), `scripts/docs/screenshots.go`.

**Depends on:** Phases 0–9 (code and docs baseline).

```markdown
# End-to-End Walkthrough

Audience: a new operator going from zero to a first published Patreon post.

## Prerequisites
- Go 1.26.1 installed (`go version`)
- Podman 4+ (`podman --version`)
- A Patreon creator account
- A GitHub (and optionally GitLab/GitFlic/GitVerse) account with a PAT

## 1. Clone and build
    git clone https://github.com/milos85vasic/My-Patreon-Manager.git
    cd My-Patreon-Manager
    go build ./...

Expected output: two binaries `patreon-manager` and `patreon-manager-server`.

## 2. Configure
    cp .env.example .env
    $EDITOR .env
Set at minimum: PATREON_ACCESS_TOKEN, GITHUB_TOKEN, LLM_API_KEY, LLM_PROVIDER.

## 3. Validate
    ./patreon-manager validate
Expected output: `configuration valid`. Common errors are listed in docs/troubleshooting/common-errors.md.

## 4. Dry-run sync
    ./patreon-manager sync --dry-run
Expected: a list of repositories that would be processed. No writes.

## 5. Real sync
    ./patreon-manager sync
Expected: per-repo progress log, audit entries emitted, a generated content file per repo.

## 6. Publish
    ./patreon-manager publish --tier=public
Expected: Patreon posts created, fingerprints stored, idempotency on re-run.

## 7. Monitor
    curl http://localhost:8080/metrics | grep patreon_sync_repos_total

## Troubleshooting
See docs/troubleshooting/faq.md.
```

## Task 7: Screenshot generation script

**Files:**
- Create: `scripts/docs/screenshots.go`
- Create: `docs/manuals/screenshots/*.png`

- [ ] **Step 1: Implement** a Go program using chromedp that:
  - spins up a local test server,
  - navigates to `/metrics`, `/health`, Grafana dashboard, SonarQube project page,
  - captures PNGs into `docs/manuals/screenshots/`.

```go
// scripts/docs/screenshots.go
package main

import (
	"context"
	"os"
	"path/filepath"

	"github.com/chromedp/chromedp"
)

type shot struct{ name, url string }

func main() {
	ctx, cancel := chromedp.NewContext(context.Background())
	defer cancel()
	shots := []shot{
		{"health", "http://localhost:8080/health"},
		{"metrics", "http://localhost:8080/metrics"},
		{"grafana", "http://localhost:3000/d/patreon-overview"},
	}
	dir := "docs/manuals/screenshots"
	_ = os.MkdirAll(dir, 0o755)
	for _, s := range shots {
		var buf []byte
		if err := chromedp.Run(ctx,
			chromedp.Navigate(s.url),
			chromedp.FullScreenshot(&buf, 90),
		); err != nil {
			continue
		}
		_ = os.WriteFile(filepath.Join(dir, s.name+".png"), buf, 0o644)
	}
}
```
