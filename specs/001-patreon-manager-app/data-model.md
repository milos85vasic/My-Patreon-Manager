# Data Model: My Patreon Manager Application

**Feature Branch**: `001-patreon-manager-app`
**Date**: 2026-04-09

## Entities

### Repository

Represents a Git repository discovered through scanning. Central entity for
the sync pipeline.

| Field | Type | Description |
|-------|------|-------------|
| ID | string (UUID) | Internal unique identifier |
| Service | string | Git service name: "github", "gitlab", "gitflic", "gitverse" |
| Owner | string | Repository owner (user or organization) |
| Name | string | Repository name |
| URL | string | Canonical SSH URL (git@host:owner/name.git) |
| HTTPSURL | string | HTTPS equivalent URL |
| Description | string | Repository description |
| READMEContent | string | README file content (truncated to 10KB) |
| READMEFormat | string | "markdown", "rst", "text" |
| Topics | []string | Tags/topics associated with the repo |
| PrimaryLanguage | string | Most used programming language |
| LanguageStats | map[string]float64 | Language breakdown as percentages |
| Stars | int | Star/favorite count |
| Forks | int | Fork count |
| LastCommitSHA | string | SHA of the most recent commit |
| LastCommitAt | timestamp | Timestamp of most recent push |
| IsArchived | bool | Whether the repository is archived |
| CreatedAt | timestamp | First discovery timestamp |
| UpdatedAt | timestamp | Last metadata refresh timestamp |

**Identity**: Unique on composite (Service, Owner, Name).

**Relationships**:
- Has one SyncState
- Has many MirrorMap entries
- Has many AuditEntry records
- Source for GeneratedContent
- Maps to one Patreon Post (via SyncState)

### SyncState

Tracks the synchronization state per repository for incremental processing.

| Field | Type | Description |
|-------|------|-------------|
| ID | string (UUID) | Primary key |
| RepositoryID | string (UUID) | FK to Repository |
| PatreonPostID | string | Patreon post ID (empty if not yet published) |
| LastSyncAt | timestamp | Last successful sync time |
| LastCommitSHA | string | Commit SHA at last sync |
| LastContentHash | string | Hash of last generated content |
| Status | string | "pending", "synced", "failed", "grace_period" |
| LastFailureReason | string | Error message if status is "failed" |
| GracePeriodUntil | timestamp | Deadline for grace period removal |
| Checkpoint | string (JSON) | Serialized checkpoint state |
| CreatedAt | timestamp | Record creation time |
| UpdatedAt | timestamp | Last status change |

**Identity**: One-to-one with Repository.

**State Transitions**:
```
pending → synced (on successful sync)
pending → failed (on sync error)
synced → pending (on detected change)
synced → grace_period (on repository removal)
grace_period → synced (on repository revival)
grace_period → archived (on grace period expiry)
failed → pending (on retry)
```

### MirrorMap

Maps repositories identified as mirrors across Git services.

| Field | Type | Description |
|-------|------|-------------|
| ID | string (UUID) | Primary key |
| MirrorGroupID | string (UUID) | Groups all mirrors of the same project |
| RepositoryID | string (UUID) | FK to Repository |
| IsCanonical | bool | Whether this is the designated primary source |
| ConfidenceScore | float64 | Mirror detection confidence (0.0-1.0) |
| DetectionMethod | string | "name_match", "readme_hash", "commit_sha", "description_similarity" |
| CreatedAt | timestamp | Detection timestamp |

**Identity**: Unique on (MirrorGroupID, RepositoryID).

### GeneratedContent

Represents content generated by the LLM pipeline for a repository.

| Field | Type | Description |
|-------|------|-------------|
| ID | string (UUID) | Primary key |
| RepositoryID | string (UUID) | FK to Repository |
| ContentType | string | "overview", "technical_doc", "sponsorship", "announcement" |
| Format | string | "markdown", "html", "pdf", "video_script" |
| Title | string | Generated title |
| Body | string (text) | Generated content body |
| QualityScore | float64 | LLMsVerifier quality score (0.0-1.0) |
| ModelUsed | string | LLM model identifier |
| PromptTemplate | string | Template name used for generation |
| TokenCount | int | Total tokens consumed |
| GenerationAttempts | int | Number of attempts (fallbacks) |
| PassedQualityGate | bool | Whether content met quality threshold |
| CreatedAt | timestamp | Generation timestamp |

**Identity**: Unique on (RepositoryID, ContentType, Format, CreatedAt).

### ContentTemplate

Prompt template with variable placeholders for content generation.

| Field | Type | Description |
|-------|------|-------------|
| ID | string (UUID) | Primary key |
| Name | string | Template identifier (e.g., "project_overview") |
| ContentType | string | "overview", "technical_doc", "sponsorship", "announcement" |
| Language | string | ISO 639-1 code (e.g., "en") |
| Template | string (text) | Prompt template with {{VARIABLE}} placeholders |
| Variables | []string | List of expected variable names |
| MinLength | int | Minimum acceptable content length |
| MaxLength | int | Maximum acceptable content length |
| QualityTier | string | "standard", "premium", "minimal" |
| IsBuiltIn | bool | Whether this is a system template |
| CreatedAt | timestamp | Creation time |
| UpdatedAt | timestamp | Last modification |

### Campaign

Represents the Patreon campaign being managed.

| Field | Type | Description |
|-------|------|-------------|
| ID | string | Patreon campaign ID |
| Name | string | Campaign name |
| Summary | string | Campaign description |
| CreatorName | string | Creator display name |
| PatronCount | int | Current patron count |
| CreatedAt | timestamp | First sync timestamp |
| UpdatedAt | timestamp | Last metadata refresh |

**Identity**: Patreon campaign ID (singleton per configuration).

### Tier

Represents a Patreon subscription tier.

| Field | Type | Description |
|-------|------|-------------|
| ID | string | Patreon tier ID |
| CampaignID | string | FK to Campaign |
| Title | string | Tier title |
| Description | string | Tier description |
| AmountCents | int | Monthly price in cents |
| PatronCount | int | Current subscriber count |
| CreatedAt | timestamp | Discovery timestamp |
| UpdatedAt | timestamp | Last refresh |

**Identity**: Patreon tier ID.

### Post

Represents a Patreon post created or managed by the system.

| Field | Type | Description |
|-------|------|-------------|
| ID | string | Patreon post ID |
| CampaignID | string | FK to Campaign |
| RepositoryID | string (UUID) | FK to Repository (source) |
| Title | string | Post title |
| Content | string (text) | Post body (HTML or Markdown) |
| PostType | string | "text", "image", "video", "link", "file" |
| TierIDs | []string | Associated tier IDs |
| PublicationStatus | string | "draft", "published", "scheduled" |
| PublishedAt | timestamp | Publication time |
| IsManuallyEdited | bool | Whether creator edited after automation |
| ContentHash | string | Hash of content at last automated update |
| CreatedAt | timestamp | Creation time |
| UpdatedAt | timestamp | Last update time |

**Identity**: Patreon post ID.

### AuditEntry

Versioned record of content lifecycle events for compliance and debugging.

| Field | Type | Description |
|-------|------|-------------|
| ID | string (UUID) | Primary key |
| RepositoryID | string (UUID) | FK to Repository |
| EventType | string | "create", "update", "archive", "delete", "generate", "publish" |
| SourceState | string (JSON) | Repository state at event time |
| GenerationParams | string (JSON) | LLM parameters (model, prompt, tokens) |
| PublicationMeta | string (JSON) | Patreon post metadata |
| Actor | string | "system", "manual", "webhook" |
| Outcome | string | "success", "failure", "skipped" |
| ErrorMessage | string | Error details if outcome is "failure" |
| Timestamp | timestamp | Event time |

**Retention**: 7 years (per constitution Principle V).

### SyncLock

Tracks active sync executions for dual-layer locking.

| Field | Type | Description |
|-------|------|-------------|
| ID | string (UUID) | Primary key |
| PID | int | Operating system process ID |
| Hostname | string | Machine hostname |
| StartedAt | timestamp | Lock acquisition time |
| ExpiresAt | timestamp | Stale detection deadline |

**Identity**: Singleton row when active.

## SQL Schema (SQLite)

```sql
-- Migration 001: Initial schema

CREATE TABLE repositories (
    id TEXT PRIMARY KEY,
    service TEXT NOT NULL,
    owner TEXT NOT NULL,
    name TEXT NOT NULL,
    url TEXT NOT NULL,
    https_url TEXT NOT NULL,
    description TEXT DEFAULT '',
    readme_content TEXT DEFAULT '',
    readme_format TEXT DEFAULT 'text',
    topics TEXT DEFAULT '[]',           -- JSON array
    primary_language TEXT DEFAULT '',
    language_stats TEXT DEFAULT '{}',    -- JSON object
    stars INTEGER DEFAULT 0,
    forks INTEGER DEFAULT 0,
    last_commit_sha TEXT DEFAULT '',
    last_commit_at DATETIME,
    is_archived BOOLEAN DEFAULT FALSE,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(service, owner, name)
);

CREATE INDEX idx_repos_service ON repositories(service);
CREATE INDEX idx_repos_owner ON repositories(owner);
CREATE INDEX idx_repos_updated ON repositories(updated_at);

CREATE TABLE sync_states (
    id TEXT PRIMARY KEY,
    repository_id TEXT NOT NULL REFERENCES repositories(id) ON DELETE CASCADE,
    patreon_post_id TEXT DEFAULT '',
    last_sync_at DATETIME,
    last_commit_sha TEXT DEFAULT '',
    last_content_hash TEXT DEFAULT '',
    status TEXT NOT NULL DEFAULT 'pending'
        CHECK(status IN ('pending', 'synced', 'failed', 'grace_period', 'archived')),
    last_failure_reason TEXT DEFAULT '',
    grace_period_until DATETIME,
    checkpoint TEXT DEFAULT '{}',        -- JSON
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(repository_id)
);

CREATE INDEX idx_sync_status ON sync_states(status);

CREATE TABLE mirror_maps (
    id TEXT PRIMARY KEY,
    mirror_group_id TEXT NOT NULL,
    repository_id TEXT NOT NULL REFERENCES repositories(id) ON DELETE CASCADE,
    is_canonical BOOLEAN DEFAULT FALSE,
    confidence_score REAL DEFAULT 0.0,
    detection_method TEXT NOT NULL,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(mirror_group_id, repository_id)
);

CREATE INDEX idx_mirror_group ON mirror_maps(mirror_group_id);

CREATE TABLE generated_contents (
    id TEXT PRIMARY KEY,
    repository_id TEXT NOT NULL REFERENCES repositories(id) ON DELETE CASCADE,
    content_type TEXT NOT NULL
        CHECK(content_type IN ('overview', 'technical_doc', 'sponsorship', 'announcement')),
    format TEXT NOT NULL
        CHECK(format IN ('markdown', 'html', 'pdf', 'video_script')),
    title TEXT DEFAULT '',
    body TEXT DEFAULT '',
    quality_score REAL DEFAULT 0.0,
    model_used TEXT DEFAULT '',
    prompt_template TEXT DEFAULT '',
    token_count INTEGER DEFAULT 0,
    generation_attempts INTEGER DEFAULT 1,
    passed_quality_gate BOOLEAN DEFAULT FALSE,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_content_repo ON generated_contents(repository_id);
CREATE INDEX idx_content_quality ON generated_contents(quality_score);

CREATE TABLE content_templates (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL UNIQUE,
    content_type TEXT NOT NULL,
    language TEXT DEFAULT 'en',
    template TEXT NOT NULL,
    variables TEXT DEFAULT '[]',          -- JSON array
    min_length INTEGER DEFAULT 100,
    max_length INTEGER DEFAULT 4000,
    quality_tier TEXT DEFAULT 'standard',
    is_built_in BOOLEAN DEFAULT FALSE,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE campaigns (
    id TEXT PRIMARY KEY,
    name TEXT DEFAULT '',
    summary TEXT DEFAULT '',
    creator_name TEXT DEFAULT '',
    patron_count INTEGER DEFAULT 0,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE tiers (
    id TEXT PRIMARY KEY,
    campaign_id TEXT NOT NULL REFERENCES campaigns(id) ON DELETE CASCADE,
    title TEXT DEFAULT '',
    description TEXT DEFAULT '',
    amount_cents INTEGER DEFAULT 0,
    patron_count INTEGER DEFAULT 0,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE posts (
    id TEXT PRIMARY KEY,
    campaign_id TEXT NOT NULL REFERENCES campaigns(id) ON DELETE CASCADE,
    repository_id TEXT REFERENCES repositories(id) ON DELETE SET NULL,
    title TEXT DEFAULT '',
    content TEXT DEFAULT '',
    post_type TEXT DEFAULT 'text',
    tier_ids TEXT DEFAULT '[]',           -- JSON array
    publication_status TEXT DEFAULT 'draft'
        CHECK(publication_status IN ('draft', 'published', 'scheduled')),
    published_at DATETIME,
    is_manually_edited BOOLEAN DEFAULT FALSE,
    content_hash TEXT DEFAULT '',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_posts_repo ON posts(repository_id);
CREATE INDEX idx_posts_status ON posts(publication_status);

CREATE TABLE sync_locks (
    id TEXT PRIMARY KEY,
    pid INTEGER NOT NULL,
    hostname TEXT NOT NULL,
    started_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    expires_at DATETIME NOT NULL
);

-- Migration 002: Audit trail

CREATE TABLE audit_entries (
    id TEXT PRIMARY KEY,
    repository_id TEXT REFERENCES repositories(id) ON DELETE SET NULL,
    event_type TEXT NOT NULL
        CHECK(event_type IN ('create', 'update', 'archive', 'delete',
                             'generate', 'publish', 'token_refresh', 'access_check')),
    source_state TEXT DEFAULT '{}',       -- JSON
    generation_params TEXT DEFAULT '{}',  -- JSON
    publication_meta TEXT DEFAULT '{}',   -- JSON
    actor TEXT NOT NULL DEFAULT 'system'
        CHECK(actor IN ('system', 'manual', 'webhook')),
    outcome TEXT NOT NULL DEFAULT 'success'
        CHECK(outcome IN ('success', 'failure', 'skipped')),
    error_message TEXT DEFAULT '',
    timestamp DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_audit_repo ON audit_entries(repository_id);
CREATE INDEX idx_audit_type ON audit_entries(event_type);
CREATE INDEX idx_audit_timestamp ON audit_entries(timestamp);
```
